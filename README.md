# Campus Share Starter

This repository contains a smart campus second-hand sharing system starter with a split stack:

- Frontend: Next.js App Router + Tailwind CSS
- Admin: pure PHP + Bootstrap + Chart.js
- Database: shared MySQL/MariaDB schema
- Captcha: `gregwar/captcha` installed through Composer

## Project Structure

- `app/`: frontend routes
- `components/`: shared frontend UI
- `lib/`: mock data and helpers
- `admin/`: PHP admin dashboard
- `database/`: schema and seed files
- `docs/`: acceptance and verification docs

## Environment Files

Use `.env.example` as the committed template for local or XAMPP runs.

Important variables:

- `NEXT_PUBLIC_APP_URL`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASS`
- `ADMIN_SESSION_NAME`
- `UPLOAD_DIR`
- `UPLOAD_PUBLIC_PATH`
- `CAPTCHA_SESSION_KEY`

For Docker-oriented defaults, copy `.env.docker.example` to `.env`.

## Docker Development

The repository includes a three-service Docker stack:

- `frontend`: Next.js dev server on `http://localhost:3000`
- `admin`: PHP/Apache admin on `http://localhost:8080/admin/login.php`
- `db`: MariaDB on host port `3307`

Start it with:

```bash
docker compose up --build
```

Notes:

- Docker ports avoid common XAMPP defaults, so both can exist on the same machine.
- The admin container runs `composer install` automatically when `vendor/autoload.php` is missing.
- The database imports `database/schema.sql` and `database/seed.sql` on first startup.
- Docker Desktop or another running Docker daemon is required before `docker compose up --build` will work on Windows.

## PHP / Admin Commands

If Composer is available on the host:

```bash
composer install
```

Lint PHP files:

```powershell
Get-ChildItem -Path admin -Recurse -Filter *.php | ForEach-Object { php -l $_.FullName }
```

If `php` is not on `PATH`, use the XAMPP PHP binary after XAMPP is installed, for example:

```powershell
C:\xampp\php\php.exe -l admin\includes\config.php
```

## Database Setup

If you are not using Docker for the database, import manually:

```bash
mysql -u root -p < database/schema.sql
mysql -u root -p < database/seed.sql
```

Important:

- Replace placeholder password hashes in `database/seed.sql` with real hashes generated by `password_hash()`.
- The PHP admin reads database settings from `.env` through `admin/includes/config.php`.
